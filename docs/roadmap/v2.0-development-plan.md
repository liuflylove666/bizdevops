# JeriDevOps v2.0 开发计划

> 从 v1.x → v2.0（BizDevOps × GitOps）。总周期 14 周 / 7 个 Sprint / 4 个里程碑。
> 本文档与 [`docs/adr/`](../adr/) 共同构成本次重构的单一真相源。

## 1. 交付目标

以 **BizDevOps 价值流**为骨架，**GitOps** 为唯一发布实现，把产品从"工具集合"升级为"端到端价值流平台"。详细产品蓝图见产品总监侧优化方案（内部 Wiki）。

## 2. 里程碑

| 里程碑 | 时间点 | 完成定义（DoD） |
|---|---|---|
| **M1 · 菜单重构 & 主线可见** | 第 2 周末 | 新菜单（V2，灰度开关控制）、工作台 DORA 卡片、旧路由 301 兼容 |
| **M2 · GitOps 发布可用** | 第 6 周末 | Release → PR → Argo CD 闭环，3 个 Pilot 应用切换完成 |
| **M3 · 应用中心聚合完成** | 第 10 周末 | 应用详情聚合页上线，历史流量治理等旧页面下线 |
| **M4 · v2.0 GA** | 第 14 周末 | 废弃能力物理删除，数据迁移完成，文档与培训到位 |

## 3. Epic 与 Sprint 排期

| Sprint | 周次 | 核心 Epic | 关键交付 |
|---|---|---|---|
| S1 | W1-W2 | Epic 1 · IA 重构 + Epic 7 前置 | 新菜单 V2、工作台 MVP、ADR 固化、Feature Flag 清单 |
| S2 | W3-W4 | Epic 2 上半 | Release 聚合根模型、PR 自动化生成器（雏形） |
| S3 | W5-W6 | Epic 2 下半 | Argo CD 注册 / Sync、风险评分、Pilot 切换 → **M2** |
| S4 | W7-W8 | Epic 3 上半 + 6.1 | 应用中心聚合骨架、成本压缩 |
| S5 | W9-W10 | Epic 3 下半 + 4 | 观测 Tab、日志/告警瘦身、应用中心完成 → **M3** |
| S6 | W11-W12 | Epic 5 + 6.2-6.4 | AI Copilot 全局化、安全合并、通知统一 |
| S7 | W13-W14 | Epic 7 收尾 + 硬化 | 遗留清理、数据迁移、性能回归、培训 → **M4** |

## 5. 风险矩阵

| 风险 | 等级 | 应对 |
|---|---|---|
| GitOps 改造影响线上发布 | 🔴 高 | Feature Flag 双路径并存 3~4 周；3 个 Pilot 先行 |
| Argo CD 学习曲线 | 🟡 中 | Sprint 6 安排 2 场培训 + 自动化 PR 模板 |
| 数据迁移风险 | 🔴 高 | 全量备份 + DRY-RUN 迁移 + 旧表保留 3 个月 |
| 功能裁剪用户反弹 | 🟡 中 | Sprint 1 前发布《变更说明》，路由 301 兼容 |
| 进度超期 | 🟡 中 | Epic 7.4 / 7.5 可延后到 v2.1 |

## 6. 质量门槛

| 维度 | 指标 |
|---|---|
| 应用详情页首屏 | ≤ 1.5s |
| 发布单 PR 生成 | ≤ 5s |
| 核心 API P99 | ≤ 500ms |
| 后端单测覆盖率 | ≥ 60%（关键服务 ≥ 80%） |
| E2E 场景 | ≥ 30 个全绿 |
| 敏感字段加密 | 100% |
